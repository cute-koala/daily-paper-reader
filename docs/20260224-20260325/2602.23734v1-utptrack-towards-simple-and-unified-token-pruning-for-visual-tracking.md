## 摘要
基于单流 Transformer 的跟踪器在视觉目标跟踪中取得了先进的性能，但面临巨大的计算开销，阻碍了其实时部署。虽然 Token 剪枝为提高效率提供了一条路径，但现有方法较为零散。它们通常孤立地对搜索区域、动态模板和静态模板进行剪枝，忽略了关键的组件间依赖关系，导致剪枝效果欠佳且精度下降。为解决这一问题，我们提出了 UTPTrack，这是一个简单且统一的 Token 剪枝框架，首次实现了对这三个组件的联合压缩。UTPTrack 采用了一种注意力引导、Token 类型感知的策略来整体建模冗余，该设计在单一模型中无缝支持跨多模态和语言引导任务的统一跟踪。在 10 个基准测试上的广泛评估表明，UTPTrack 在剪枝类跟踪器的准确率-效率权衡方面达到了新的最先进水平，在基于 RGB 的跟踪中剪枝了 65.4% 的视觉 Token，在统一跟踪中剪枝了 67.5%，同时分别保留了 99.7% 和 100.5% 的基准性能。在 RGB 和多模态场景下的强劲表现突显了其作为高效视觉跟踪未来研究稳健基础的潜力。代码将发布在 https://github.com/EIT-NLP/UTPTrack。

## Abstract
One-stream Transformer-based trackers achieve advanced performance in visual object tracking but suffer from significant computational overhead that hinders real-time deployment. While token pruning offers a path to efficiency, existing methods are fragmented. They typically prune the search region, dynamic template, and static template in isolation, overlooking critical inter-component dependencies, which yields suboptimal pruning and degraded accuracy. To address this, we introduce UTPTrack, a simple and Unified Token Pruning framework that, for the first time, jointly compresses all three components. UTPTrack employs an attention-guided, token type-aware strategy to holistically model redundancy, a design that seamlessly supports unified tracking across multimodal and language-guided tasks within a single model. Extensive evaluations on 10 benchmarks demonstrate that UTPTrack achieves a new state-of-the-art in the accuracy-efficiency trade-off for pruning-based trackers, pruning 65.4% of vision tokens in RGB-based tracking and 67.5% in unified tracking while preserving 99.7% and 100.5% of baseline performance, respectively. This strong performance across both RGB and multimodal scenarios underlines its potential as a robust foundation for future research in efficient visual tracking. Code will be released at https://github.com/EIT-NLP/UTPTrack.

### UTPTrack：面向视觉跟踪的简单统一 Token 剪枝框架论文总结

#### 1. 核心问题与整体含义（研究动机和背景）
基于单流 Transformer 的跟踪器（如 OSTrack）虽然性能卓越，但由于 Transformer 的二次方计算复杂度以及大量的视频 Token，导致其计算开销巨大，难以在资源受限的设备上实时部署。
*   **现有局限：** 现有的 Token 剪枝方法通常是“碎片化”的，仅孤立地对搜索区域（SR）或动态模板（DT）进行剪枝，忽略了搜索区域、动态模板和静态模板（ST）之间的内在依赖关系。
*   **研究动机：** 这种孤立的剪枝策略会导致冗余建模不充分，可能误删关键信息，尤其在多模态场景下表现不佳。因此，论文旨在提出一个**统一的剪枝框架**，同时压缩所有关键组件。

#### 2. 方法论：核心思想与关键技术
UTPTrack 的核心思想是在单流架构中通过**注意力引导**和**Token 类型感知**策略，联合剪枝 SR、DT 和 ST。
*   **联合剪枝框架：** 在 Transformer 编码器层中插入轻量级的候选或模板消除模块（CTEM）。
*   **注意力引导剪枝：** 利用 Transformer 固有的注意力权重计算 Token 重要性。具体而言，计算各 Token 与静态模板中心 Token 的相似度，保留得分最高的 Token。
*   **Token 类型感知（TTA）策略：** 针对静态模板（ST），利用初始边界框的先验空间信息，为前景区域的 Token 提供“奖励分”（Bonus），防止在剪枝过程中误删目标核心特征。
*   **统一多模态扩展：** 
    *   **视觉模态：** 在共享嵌入空间中对 RGB、深度、热成像、事件等模态进行统一剪枝。
    *   **文本引导剪枝：** 在 RGB-语言任务中，引入文本 Token 的语义线索来共同引导视觉 Token 的筛选。

#### 3. 实验设计
论文在 **10 个主流基准测试**上进行了评估，涵盖了 RGB 跟踪和多模态跟踪：
*   **RGB 跟踪数据集：** LaSOT, LaSOT ext, TrackingNet, GOT-10k。
*   **多模态跟踪数据集：** VOT-RGBD22 (深度), LasHeR (热成像), RGBT234 (热成像), VisEvent (事件), TNL2K (语言), OTB99 (语言)。

*   **评估协议：** 采用了“默认设置性能”和“受控预算性能”（在相同剪枝比例下对比）两种协议，确保公平性。

*   **训练算力：** 使用了 **4 台 NVIDIA A100 GPU** 进行训练。

#### 5. 实验数量与充分性
*   **实验规模：** 论文进行了极其详尽的实验，包括 10 个数据集的性能对比、不同分辨率（224/256/384）的测试。
*   **消融实验：** 针对 SR/DT/ST 各组件的剪枝贡献、TTA 策略的不同奖励方式、语言线索的注入位置、剪枝发生的层数位置等做了深入分析。
*   **客观性：** 通过“受控预算”实验，排除了因剪枝比例不同导致的性能偏差，证明了在相同计算量下 UTPTrack 的策略优于其他通用剪枝方法。

#### 6. 主要结论与发现
*   **高效压缩：** 在 RGB 跟踪中剪枝了 **65.4%** 的 Token，在统一跟踪中剪枝了 **67.5%**，而性能分别保留了基准的 **99.7%** 和 **100.5%**。
*   **性能提升：** 在某些高分辨率场景下，剪枝后的模型性能甚至超过了未剪枝的基准，这表明剪枝起到了一定的正则化作用，去除了背景噪声。
*   **多模态通用性：** 证明了简单的注意力引导机制足以处理复杂的跨模态对齐和冗余消除。

#### 7. 优点（亮点）
*   **首创性：** 首次实现了对搜索区域、动态模板和静态模板的**联合剪枝**。
*   **简单高效：** 无需引入额外的预测模块或复杂的启发式规则，直接复用模型内部的注意力图，计算开销极低。
*   **高度统一：** 一个模型、一套策略即可处理 RGB 及多种辅助模态（深度、热、事件、语言）。
*   **即插即用：** 架构无关，可轻松集成到其他单流 Transformer 跟踪器中。

#### 8. 不足与局限
*   **长时语言跟踪退化：** 在 TNL2K 等长视频语言引导任务中，性能相比基准有轻微下降，可能是因为长序列中初始语言描述的指导作用随时间减弱，而剪枝进一步放大了这种不确定性。
*   **依赖初始框：** TTA 策略依赖于第一帧的边界框质量，如果初始框不准，可能会影响整个序列的静态模板剪枝效果。
*   **硬件加速限制：** 虽然减少了 MACs（计算量），但在某些 GPU 架构上，由于 Token 数量不规则导致的内存访问开销，实际 FPS 的提升可能不如计算量减少得那么显著（尽管文中已通过 Zero-padding 缓解）。
