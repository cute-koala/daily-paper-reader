## 摘要
我们研究了时间层次结构是否能改善LeWorldModel在长时域目标条件控制中的表现。我们引入Hi-LeWM，一个扩展模型，冻结预训练的低层LeWM，并添加对潜在子目标的高层规划。我们在PushT和Cube任务上评估Hi-LeWM，增加目标偏移量。层次结构并不会自动提升性能：在短时域下，最佳配置使用单步高层时域，而长时域则暴露出学习到的高层动作空间与推理时搜索分布之间的不匹配。使用真实未来潜在子目标的实验表明，冻结的低层控制器可以良好执行对齐的中间目标，表明高层子目标生成是主要瓶颈。无约束搜索可能选择在所学模型下看似有利但产生不良控制目标的潜在宏观动作。通过对从训练轨迹编码的宏观动作进行约束搜索，并配以适当的子目标执行时机，可以恢复有用的层次化机制，在中程时域上比平坦LeWM提升11.3个百分点，在最长PushT时域上提升14.7个百分点。总体而言，时间抽象可以有益于紧凑的冻结LeWM，但前提是高层搜索与低层控制器保持兼容。

## Abstract
We investigate whether temporal hierarchy can improve LeWorldModel on long-horizon goal-conditioned control. We introduce Hi-LeWM, an extension that freezes the pretrained low-level LeWM and adds high-level planning over latent subgoals. We evaluate Hi-LeWM on PushT and Cube across increasing goal offsets. Hierarchy does not automatically improve performance: at short horizons, the best configuration uses a one-step high-level horizon, while longer horizons reveal a mismatch between the learned high-level action space and the inference-time search distribution. Experiments with true future latent subgoals show that the frozen low-level controller can execute well-aligned intermediate targets, indicating that high-level subgoal generation is the main bottleneck. Unconstrained search can select latent macro-actions that appear favorable under the learned model but produce poor control targets. Constraining search around macro-actions encoded from training trajectories, with appropriate subgoal execution timing, recovers useful hierarchical regimes, improving over flat LeWM by +11.3 percentage points at medium-range horizons and +14.7 percentage points at the longest PushT horizon. Overall, temporal abstraction can benefit compact frozen LeWM, but only when high-level search remains compatible with the low-level controller

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：长时域目标条件控制在机器人学习中是重要挑战。LeWorldModel (LeWM) 是一种紧凑的、基于JEPA风格的潜在世界模型，使用交叉熵方法（CEM）在潜在空间进行规划。虽然LeWM在短时域上表现良好，但随着规划时域增长，模型误差累积、行动序列优化变得困难，导致性能急剧下降。
- **核心问题**：能否通过添加时间层次结构（即高层规划器生成潜在子目标，低层LeWM执行）来改善长时域控制？本文旨在探究“简单的分层扩展”是否适用于紧凑的LeWM，并识别其成功条件与失败瓶颈。
- **整体含义**：时间抽象（temporal abstraction）可以改善紧凑冻结的世界模型，但只有在高层搜索空间与低层控制器兼容时才有效。约束高层搜索（使用数据支持的宏动作）是关键的补救措施。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：提出Hi-LeWM，保持预训练的LeWM低层组件（编码器、低层预测器、低层行动编码器）冻结，新增一个高层模块：一个宏动作编码器 $g$ 和一个高层预测器 $\phi^{\text{hi}}$。高层模块学习潜在宏动作（latent macro-actions）和抽象状态转移。
- **关键技术细节**：
    - **宏动作编码**：将两个路径点之间的原始行动块（primitive action chunk）编码为固定维度的潜在宏动作 $\ell_i \in \mathbb{R}^{d_\ell}$（默认 $d_\ell=32$）。编码器是两层Transformer。
    - **高层预测器**：基于当前路径点潜在状态 $z_{\tau_i}$ 和宏动作 $\ell_i$，预测下一路径点潜在状态 $\hat{z}_{\tau_{i+1}} = \phi^{\text{hi}}(z_{\tau_i}, \ell_i)$。训练损失为预测与编码的真实未来状态之间的均方误差（MSE）。
    - **规划流程**：测试时，高层CEM在宏动作空间中搜索宏动作序列，最大化终端预测与目标状态的相似度；第一个预测的子目标被传递给冻结的低层LeWM规划器，低层以标准MPC方式朝该子目标推理。
    - **约束搜索（Empirical-macro CEM）**：为解决无约束搜索产生不良子目标的问题，构建一个由训练轨迹编码的宏动作序列库，在CEM中围绕该库的锚点进行局部残差扰动采样，使搜索保持数据支持。
    - **执行模式**：比较在线重规划和阶段式执行（staged execution，仅执行一次高层计划，依次切换子目标）。
- **算法流程**（文字说明）：
    1. 冻结LeWM低层（编码器 $f_\theta$、低层预测器、低层行动编码器）。
    2. 训练高层：从训练轨迹中采样路径点对，将行动块编码为宏动作，训练高层预测器最小化 $\| \hat{z}_{\tau_{i+1}} - z_{\tau_{i+1}} \|^2$。
    3. 测试时，高层CEM搜索宏动作序列（无约束或empirical-macro），选择使终端潜在状态最接近目标潜在状态的序列。
    4. 将第一个子目标传入低层LeWM，低层MPC规划原始行动执行前缀，然后重规划。
    5. 阶段式执行：高层只在开始时运行一次，将子目标列表按固定阶段长度提供给低层。

## 6. 论文的主要结论与发现

- **分层不自动改善性能**：在短时域（$d=25$）平坦LeWM已接近饱和，Hi-LeWM无增益；在中长时域（$d=50，75$），标准高层CEM的Hi-LeWM甚至不如平坦LeWM（$d=50$时低14.0%成功率）。
- **主要瓶颈是高层搜索空间不匹配**：无约束高层CEM可以选择在预测末端损失很低但中间子目标不可执行的宏动作（支撑外搜索，out-of-support search）。诊断实验（表1）显示，CEM选择的宏动作第一段预测误差大，但第二段误差小，说明CEM利用了终端损失进行“投机取巧”。
- **Oracle子目标证明低层可执行**：使用真实未来潜在状态作为子目标时，成功率较高（$d=50$时73.3% vs 生成子目标的38.7%），说明低层控制器本身有能力，问题在于子目标生成质量。
- **约束搜索有效**：Empirical-macro CEM通过将高层搜索限制在数据支持的宏动作附近，显著提升性能。最佳在线Hi-LeWM-C在$d=50$达到48.7%，$d=75$达到32.7%。
- **阶段式执行在中程时域有效**：阶段式执行（不重规划高层）在$d=50$取得64.0%，超越平坦LeWM 11.3个百分点；但在$d=75$阶段式效果差（22.0%），在线约束更好（32.7%）。说明阶段式需要高层计划可靠保持较长时间才有效。
- **最终结论**：时间抽象可以有利于紧凑冻结LeWM，但前提是高层搜索空间与低层兼容，且执行模式需匹配任务时域尺度。

## 7. 优点

- **清晰的诊断链**：通过Oracle实验和预测误差分解（表1），精准定位失败原因（支撑外搜索）而非笼统归因于模型能力不足。
- **最小化改动**：只添加高层模块，冻结低层LeWM，隔离了层次结构本身的影响，避免了改变整个训练流程带来的混淆。
- **实用性的改进方案**：Empirical-macro CEM是一种无需重新训练即可提升性能的推理时干预，易于在实际系统中使用。
- **开源与可复现**：提供代码和checkpoints，实验构建在stable-worldmodel库上，保证了可复现性。
- **充分的超参数扫描**：对多个规划超参数进行系统扫描，并报告最佳配置，避免了调参偏向。

## 8. 不足与局限

- **环境多样性不足**：核心实验仅使用PushT一个环境，Cube因平坦基线稳定而诊断性弱。结论在更复杂或更长时域任务中的泛化性未知。
- **冻结低层是双刃剑**：虽然隔离了层次影响，但也限制了高层-低层联合优化可能带来的更大收益。未探索端到端联合训练能否缓解搜索不匹配问题（虽然文中提及联合训练未带来提升，但未详细报告）。
- **阶段式执行的局限性**：在最长时域（$d=75$）阶段式失败，论文未深入原因，可能因为整个高层序列可靠性不足，或对阶段长度的敏感度未研究。
- **宏动作表示实验未系统融入核心分析**：VQ宏动作在$d=75$有提升（34% vs 20%），但论文选择不作为主要变体，因为改变了架构。这限制了关于离散化是否更鲁棒的结论。
- **计算开销未详细对比**：虽然报告了训练时间，但未比较平坦LeWM与Hi-LeWM在推理时规划延迟或计算量的差异。
- **目标偏移$d$变化范围有限**：最大150步，更长时域（如200+）的行为未知，下界$d=25$时平坦已饱和，未在更短或更困难的情境下测试。
