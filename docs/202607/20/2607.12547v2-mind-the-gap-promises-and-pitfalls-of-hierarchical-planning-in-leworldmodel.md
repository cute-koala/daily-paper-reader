## 摘要
我们研究时间层次结构是否能改善长期目标条件控制中的LeWorldModel。我们引入Hi-LeWM，这是一种扩展方法，冻结预训练的低层LeWM，并增加对潜在子目标的高层规划。我们在PushT和Cube任务上评估Hi-LeWM，不断增加目标偏移量。层次结构并不能自动提升性能：在短时间范围内，最佳配置使用一步高层规划，而较长时间范围则揭示了所学高层动作空间与推理时搜索分布之间的不匹配。使用真实未来潜在子目标的实验表明，冻结的低层控制器可以很好地执行对齐的中间目标，表明高层子目标生成是主要瓶颈。无约束的搜索可以选择在所学模型下看似有利但产生糟糕控制目标的潜在宏观动作。将搜索约束在从训练轨迹编码的宏观动作周围，并适当执行子目标，可以恢复有用的层次机制，在中等范围的时间范围内比平坦LeWM提高了+11.3个百分点，在最长的PushT时间范围内提高了+14.7个百分点。总体而言，时间抽象可以有益于紧凑的冻结LeWM，但仅当高层搜索与低层控制器保持兼容时。

## Abstract
We investigate whether temporal hierarchy can improve LeWorldModel on long-horizon goal-conditioned control. We introduce Hi-LeWM, an extension that freezes the pretrained low-level LeWM and adds high-level planning over latent subgoals. We evaluate Hi-LeWM on PushT and Cube across increasing goal offsets. Hierarchy does not automatically improve performance: at short horizons, the best configuration uses a one-step high-level horizon, while longer horizons reveal a mismatch between the learned high-level action space and the inference-time search distribution. Experiments with true future latent subgoals show that the frozen low-level controller can execute well-aligned intermediate targets, indicating that high-level subgoal generation is the main bottleneck. Unconstrained search can select latent macro-actions that appear favorable under the learned model but produce poor control targets. Constraining search around macro-actions encoded from training trajectories, with appropriate subgoal execution timing, recovers useful hierarchical regimes, improving over flat LeWM by +11.3 percentage points at medium-range horizons and +14.7 percentage points at the longest PushT horizon. Overall, temporal abstraction can benefit compact frozen LeWM, but only when high-level search remains compatible with the low-level controller

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：长期视界规划（long-horizon planning）是潜在世界模型（latent world model）面临的严峻挑战。LeWorldModel (LeWM) 作为一种紧凑的 JEPA 风格模型，使用 CEM 在潜在空间进行平坦规划（flat planning），但模型误差在长期想象 rollout 中积累，且直接优化原始动作序列的复杂度随规划视界增长而急剧上升。
- **背景**：时间抽象（temporal abstraction）被广泛认为是解决长期规划的有效方向。已有工作（如 THICK、Dreamer、Hierarchical DINO-WM）引入层次化世界模型并报告了性能提升，但这些方法通常改变了模型架构、训练流程或规划接口。本研究希望作为 **轻量级 retrofit**，在 **保持低层 LeWM 冻结** 的前提下，仅增加高层组件，考察“层次结构是否自动提升紧凑 LeWM 的长期规划能力”。

## 2. 论文提出的方法论：核心思想与关键技术细节

- **核心思想**：提出 **Hi-LeWM**，它是一种层次化扩展，冻结预训练的 LeWM 低层（编码器、低层预测器 plo、低层动作编码器），并新增两个高层组件：
  - **潜在宏动作编码器 g**：基于 Transformer，将两个连续方式点（waypoint）之间的原始动作块编码为固定维度的潜在宏动作 ℓ ∈ ℝ³²。
  - **高层预测器 φ_h**：基于带有自适应层归一化的 Transformer，输入当前方式点潜在 z_t 和宏动作 ℓ_t，预测下一个方式点潜在 ẑ_{t+k}（k 为时隙间隔）。
- **训练**：使用随机方式点采样（N=5，跨度 ≤15 步），损失函数为高层潜在预测的均方误差（MSE）。低层组件全部冻结，仅优化 g 和 φ_h。
- **推理**：采用两层规划：
  - **高层 CEM**：在宏动作序列空间搜索最佳宏动作序列（视界为 h_h），以最小化最终潜在距目标潜在的距离。选出的第一个子目标传给低层。
  - **低层 CEM**：原 LeWM 规划器，以接收到的潜在子目标为导向，优化短原始动作序列执行。
- **关键改进**：发现无约束高层 CEM 会搜索到训练数据分布外的宏动作，产生不合理子目标。为此提出 **经验宏动作 CEM (empirical-macro CEM)**：从训练轨迹中构建宏动作编码库（bank），在规划时每次随机采样一个锚序列，然后在锚周围施加小随机扰动作为候选，CEM 仅优化扰动分布，确保搜索始终贴近训练支持范围。此外，还探索了 **分段执行**（staged execution）模式，即高层一次性规划出完整子目标序列，然后低层依次执行，不再在线重规划。

## 6. 论文的主要结论与发现

1. **短视界**：平坦 LeWM 已经饱和（d=25 时成功率 94%），层次结构无优势，且最佳 Hi-LeWM 配置使用 h_h=1 退化为一步子目标规划器。
2. **中长视界**：无约束 Hi-LeWM 表现差于平坦基线（d=50 时低 14%）；失败根源是 **高层 CEM 搜索到训练支持范围外的宏动作**，产生规划成本低但实际不可执行的子目标。
3. **约束搜索有效**： empirical-macro CEM 结合 **分段执行** 在中视界（d=50）提升 +11.3 百分点（64.0% vs 52.7%）；在最视界（d=75）分段执行反而更差，但 **在线约束重规划** 提升 +14.7 百分点（32.7% vs 18.0%）。
4. **时间尺度敏感性**：分段执行能否帮助取决于高层生成序列在承诺区间内是否可靠；最长视界下即使约束搜索也无法保证，故需在线修正。

## 7. 优点

- **轻量级 retrofit**：冻结低层，仅增加高层模块，无需重新训练完整模型，计算成本低，易于复用。
- **系统诊断**：通过 oracle 替换、预测误差对比等实验，清晰定位了失败链中的关键瓶颈——搜索分布支持不匹配，而非低层可执行性或高层模型预测能力。
- **可复现性**：提供了代码和检查点（Zenodo），并使用 `stable-worldmodel` 库确保接口一致性。
- **分析深度**：不仅报告性能，还从“支持匹配”视角解释了层次结构为何有时有害，并提出有针对性的解决方案。

## 8. 不足与局限

- **环境局限性**：主要实验在 PushT 上，OGBCube 因长视界下平坦基线未显著退化，不具诊断意义。能否推广到更复杂、更开放的任务（如导航、机器人操作）未知。
- **未探索联合训练**：保持低层冻结是故意设定，但作者也承认若允许联合微调低层，可能改变结论。VQ 等变体虽好但引入了额外复杂度，未完全分离架构选择与搜索约束的影响。
- **分段执行的时间尺度适应问题**：在最长视界下分段执行失败，说明需要自适应决定何时采用分段、何时在线重规划，文中未给出通用策略。
- **模型容量混淆**：对比 DINO-WM（更高容量），LeWM 获得更弱结果，因此发现可能部分受限于模型容量，而非层次结构本身局限性。
- **不完整超参调优**：尽管进行了扫描，但 CEM 参数（budget、elites 等）为离散选择，可能遗漏最优组合。
