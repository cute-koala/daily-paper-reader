## 摘要
现代视频对象分割（VOS）涉及跟踪和分割用户指定的目标。尽管近期方法在单目标场景中取得了显著性能，但将其扩展到多目标设置通常需要对每个单独对象复制单目标处理流程，导致帧率（FPS）下降，且延迟随目标数量增加而无限制增长。基于Segment Anything 2（SAM2），我们提出SAM-MT，通过将模型转化为一个用于实时多目标视频分割的交互式框架来解决这一问题。SAM-MT使用显式查询来表示不同的单个目标，同时使用共享表示来捕捉全局上下文。它采用解耦掩码注意力来保持个体身份免受跨目标干扰，利用稀疏记忆实现稳定的时间演化，并配备专门策略处理遮挡和防止重叠。SAM-MT成功地将延迟与目标数量解耦，实现了与单目标基线相当的实时速度（10个目标时>36 FPS），同时保持了SAM2鲁棒的视频分割性能。

## Abstract
Modern Video Object Segmentation (VOS) involves tracking and segmenting user-specified targets. While recent approaches have achieved remarkable performance in single-target scenarios, extending them to multi-target settings typically involves replicating the single-target processing for each individual object, resulting in reduced frame rates (FPS) with unbounded latency as target count increases. Built upon Segment Anything 2 (SAM2), we propose SAM-MT, which addresses this by transforming the model into an interactive framework for real-time Multi-Target video segmentation. SAM-MT uses explicit queries to represent different individual targets, in parallel with a shared representation for global context. It employs decoupled masked attention to keep individual identities distinct from cross-target interference, and sparse memory for stable temporal evolution, along with specialized strategies for occlusion handling and overlap prevention. SAM-MT successfully decouples latency from the number of targets, achieving real-time speed on par with single-target baselines (>36 FPS for 10 targets) while maintaining SAM2's robust video segmentation performance.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有视频对象分割（VOS）方法，如SAM2、STM、Cutie等，在单目标场景中性能优异，但扩展至多目标时，通常需要为每个目标重复执行单目标处理流程（独立编码、解码、记忆存储），导致计算量与延迟随目标数量线性增长，无法满足实时应用需求（如自动驾驶中密集人群或车辆场景）。
- **背景**：虽然部分方法（如STCN、XMem）通过共享图像特征部分降低冗余，但仍需独立处理每个目标的掩码解码和记忆编码。直接将所有目标合并为一个掩码的方式会丢失个体身份且难以跟踪。因此，需要一种既能保持个体身份、又能实现近单目标效率（延迟与目标数解耦）的多目标实时分割框架。

## 2. 论文提出的方法论

### 2.1 核心思想
- 基于SAM2架构进行扩展，引入**显式目标查询（target queries）** 表示每个单独目标，同时保留**全局上下文共享表示（global context）**，将计算成本与目标数量解耦，实现近单目标效率。

### 2.2 关键技术细节
- **解耦掩码注意力（Decoupled Masked Attention）**：在解码器的自注意力模块中设计注意力掩码M，阻止不同目标查询之间相互交互，但允许所有目标查询与全局查询交互。公式见论文式(2)-(4)，确保个体身份不因跨目标干扰而混淆。
- **目标查询合并（Query Consolidation，仅初始帧）**：对每个目标多个点击点生成的查询，通过轻量MLP权重头聚合为单个“目标查询”（式5），后续帧直接使用传播更新的单查询。
- **查询基稀疏记忆（Query-based Sparse Memory）**：仅对全局上下文保留密集记忆，对每个目标仅存储其紧凑查询（一个C维向量），以FIFO队列维护历史T帧的所有目标查询（维度T·k×C），大幅降低存储成本，支持更长时序窗口。
- **身份变换器（Identity Transformer）**：利用稀疏记忆，通过身份感知掩码（Mask ˜M，式8）使当前帧目标查询仅交叉关注自身历史查询，实现身份一致性传播，避免漂移。

### 2.3 算法流程（文字说明）
1. **初始帧**：用户提供带ID的点点击 → 编码为查询，与全局查询拼接 → 经解耦掩码自注意力 → 交叉关注图像特征 → 对每个目标聚合为目标查询 → 存入稀疏记忆。
2. **后续帧**：从稀疏记忆中检索各目标历史查询 → 身份变换器更新目标查询 → 输入解码器（同初始帧步骤，但无需合并） → 预测掩码 → 更新稀疏记忆。
3. **训练策略**：两阶段（先静态图像预训练，再视频序列训练）；跨步采样（联合连续帧和4帧步长，覆盖32帧窗口）；重叠损失（式9）惩罚不同目标预测掩码重叠。

## 6. 论文的主要结论与发现

- **性能**：SAM-MT在零样本交互设置下，在MOSEv2、LVOSv2/v1等基准上取得最佳或次佳J&F，超越SAM2.1-B+和Cutie等专用方法。例如MOSEv2 J&˙F达到43.0，比SAM2.1-B+提高1.9点；LVOSv2 J&F提高2.0点。
