## 摘要
多目标跟踪的难度分布呈现重尾特性：大多数帧对于轻量级基础跟踪器来说很容易，而一小部分帧则本质上困难。视频目标分割（VOS）模型通常能够在基础跟踪器失败的困难帧中保持身份，但它们在计算和内存方面代价更高。我们提出选择性掩码传播，这是一种跟踪算法，仅在分配不确定性信号触发的窗口上从基础跟踪器调度到VOS模型。只有当VOS模型做出与基础跟踪器身份分配相矛盾的置信预测时，基础跟踪器的输出才会被修改；微弱或不确定的预测则保留基础输出。该方法无需训练，将基础跟踪器和VOS模型均视为黑盒，并且可以通过用更强大的模型替换VOS组件而受益。在DanceTrack上，选择性掩码传播改进三种不同的基础跟踪器。在SportsMOT上，身份保持是体育分析的核心，采用全局轨迹关联的SAM3-Deep-EIoU以86.8 HOTA在基准测试中达到最先进性能。

## Abstract
Multi-object tracking has a heavy-tailed difficulty distribution: most frames are easy for a lightweight base tracker, while a small fraction are intrinsically hard. Video object segmentation (VOS) models can often preserve identity through the hard frames where the base tracker fails, but they are much more expensive in compute and memory. We propose selective mask propagation, a tracking algorithm that dispatches from a base tracker to a VOS model only on windows where an assignment-uncertainty signal fires. The base tracker's output is modified only when the VOS model makes a confident prediction that contradicts the base tracker's identity assignment; weak or inconclusive predictions preserve the base output. The method is training-free, treats both the base tracker and the VOS model as black boxes, and can benefit from replacing the VOS component with a more capable model. On DanceTrack, selective mask propagation improves three different base trackers. On SportsMOT, where identity preservation is central to sports analytics, SAM3-Deep-EIoU with global track association achieves state-of-the-art performance on the benchmark with 86.8 HOTA.

## 1. 论文的核心问题与整体含义（研究动机和背景）

多目标跟踪（MOT）的帧难度分布具有重尾特性：大多数帧可以被轻量级基础跟踪器轻松处理，但少数困难帧（如遮挡、近距离交互）容易导致身份分配错误（ID Switch）。视频目标分割（VOS）模型（如SAM系列、Cutie）能通过像素级掩码传播在困难帧中保持身份，但计算和内存开销大，且逐帧使用会导致掩码漂移、退化等问题。因此，需要一种能在保留轻量跟踪器效率的同时，仅在必要时调用VOS模型来纠正身份错误的方法。

## 2. 论文提出的方法论

### 核心思想：选择性掩码传播（Selective Mask Propagation）

- **分工**：基础跟踪器处理大多数简单帧；VOS模型仅在“风险窗口”中被调度，用于修正身份分配。
- **调度信号**：基于匈牙利分配成本的**边际差（margin）**：对每个匹配的检测，其边际差定义为第二优分配成本与最优分配成本之差。边际差小（低于阈值 τ_entry）表示分配模糊，触发窗口开启。
- **窗口创建**：除了边际信号，还有**间隙信号**（轨迹消失又重现时）、**伴随窗口**（与主窗口重叠的近邻轨迹，防止身份互换）。
- **种子帧选择**：从入口帧回溯寻找连续 N_seed 帧，要求边际高且空间孤立，确保VOS获得清晰初始掩码。
- **掩码传播与退出条件**：VOS从种子帧向前传播至窗口；在入口帧后逐帧检查是否满足所有退出条件（掩码被单个框高比例包含、该框边际恢复、框和掩码均孤立），持续 N_exit 帧后退出。
- **非对称决策规则**：只在VOS预测自信且与基础跟踪器矛盾时（掩码最终落在不同轨迹上）才修改输出（SWAP）；其余情况（掩码返回原轨迹、退化、到达边缘等）保留基础输出。这限制了相对失效模式（只有假阳性SWAP会降低性能）。

### 关键技术细节

- 使用交并比（IoMA）衡量掩码与框的重叠：`IoMA(M,B)=|M∩B|/|M|`
- 窗口结果类型：SWAP（修正身份）、CLEAN（确认原身份）、STALE（入口帧掩码已不在原框）、DEGRADED（掩码退化）、EDGE（到达图像边缘）、END（序列结束）。除SWAP外均不修改基础输出。
- 对于体育场景，额外加入**全局轨迹关联（GTA）**，利用球衣号码识别、队伍分类和外观嵌入，在离线阶段链接因离开视野而断开的轨迹。
