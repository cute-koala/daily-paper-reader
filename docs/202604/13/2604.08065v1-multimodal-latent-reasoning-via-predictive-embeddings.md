## 摘要
工具增强的多模态推理使视觉语言模型能够通过与外部工具交互来改进感知能力。然而，此类方法会产生大量推理开销、需要专门的监督，并且容易产生错误的工具调用。我们提出了Pearl，一个受JEPA启发的框架，它完全在潜在空间中学习专家工具使用轨迹，从而在推理时无需显式调用工具。与基于重建的潜在推理方法不同——后者自回归地生成潜在标记并遭受训练-推理不匹配以及对多步工具使用的支持有限——Pearl直接从多模态轨迹中学习预测性嵌入，同时保留了标准的视觉语言生成流程：它不依赖于特定模型、训练简单，并且天然支持包含多个工具调用的轨迹。在多个感知基准上的实验表明，Pearl达到或超越了标准监督微调以及基于重建的潜在推理方法的性能。此外，我们提供的经验证据表明，基于重建的方法主要学习的是嵌入而非潜在空间中的图像编辑，这促使我们将预测性嵌入学习作为一种更具原则性的替代方案。

## Abstract
Tool-augmented multimodal reasoning enables visual language models (VLMs) to improve perception by interacting with external tools (e.g., cropping, depth estimation). However, such approaches incur substantial inference overhead, require specialized supervision, and are prone to erroneous tool calls. We propose Pearl (Predictive Embedding Alignment for Reasoning in Latent space), a JEPA-inspired framework that learns from expert tool-use trajectories entirely in the latent space, eliminating the need for explicit tool invocation at inference time. Unlike reconstruction-based latent reasoning methods, which autoregressively generate latent tokens and suffer from training-inference mismatch and limited support for multi-step tool use, Pearl directly learns predictive embeddings from multimodal trajectories while preserving the standard vision-language generation pipeline: it is model-agnostic, simple to train, and naturally supports trajectories with multiple tool calls. Experiments across multiple perception benchmarks show that Pearl matches or outperforms standard supervised fine-tuning and reconstruction-based latent reasoning approaches. Furthermore, we provide empirical evidence that reconstruction-based methods primarily learn embeddings rather than image edits in latent space, motivating predictive embedding learning as a more principled alternative.

## 1. 核心问题与整体含义
*   **研究动机**：当前，通过调用外部视觉工具（如裁剪、深度估计）来增强视觉语言模型的多模态推理能力，存在三大痛点：
    1.  **高推理开销**：每次调用工具都会引入显著的延迟和计算成本。
    2.  **依赖专门监督**：学习如何正确选择和参数化工具需要专门的轨迹数据。
    3.  **错误调用风险**：错误的工具调用会浪费计算资源并污染上下文信息。
*   **研究背景**：为解决上述问题，出现了“潜在推理”方法，让模型在连续的嵌入空间中操作，而非生成离散的中间输出。其中，“基于重建的潜在推理”方法通过自回归生成潜在标记来重建工具输出图像。然而，这类方法存在**训练-推理不匹配**（训练时使用大量潜在标记，推理时只解码少量）且对**多步工具调用支持有限**。
*   **整体含义**：本文提出了一种新的范式——直接学习预测性嵌入，旨在让模型在训练阶段就将专家工具使用轨迹的效果“内化”到潜在表示中，从而在推理时完全避免显式工具调用，实现高效、鲁棒的多模态推理。

## 2. 提出的方法论
*   **核心思想 (PEARL)**：受联合嵌入预测架构启发，将原始图像-问题对 `<I0, Q>` 和专家多模态推理轨迹 `R`（包含中间编辑图像和文本）视为同一问题的两个“视图”。模型的目标是直接从输入视图预测出整个轨迹视图的潜在嵌入表示。
*   **关键技术细节**：
    1.  **双视图编码**：使用预训练的VLM分别对输入 `<I0, Q>` 和完整轨迹 `R` 进行独立前向传播，获取各自的最终隐藏状态作为序列表示 `h_x` 和 `h_R`。
    2.  **绑定权重的预测器**：在输入序列后附加 K 个可学习的 `[PRED]` 标记。利用VLM自身的注意力层对这些标记进行计算，将最后一个预测器标记的隐藏状态作为预测的轨迹嵌入 `ĥ_R`。这避免了引入额外的网络模块。
    3.  **三重损失函数**：
        *   `L_VLM`: 标准的自回归文本生成损失，确保模型保留回答问题的能力。
        *   `L_JEPA`: JEPA风格的对齐损失（使用SmoothL1），最小化预测嵌入 `ĥ_R`与停止梯度的目标嵌入 `h_R`之间的距离。这是核心的预测学习信号。
        *   `L_NextLat`: “下一潜在状态预测”损失（灵感来自信念状态学习），鼓励隐藏状态能够预测未来的轨迹演化，使其成为更好的序列摘要表示。
    4.  **总损失**: `L_PEARL = L_VLM + λ [L_JEPA + L_NextLat]`
*   **算法流程简述**: 
    *   **训练**: 
        1.  编码输入得到 `h_x`；编码专家轨迹得到目标 `h_R`。
        2.  通过绑定权重的预测器从 `h_x`得到预测嵌入 `ĥ_R`。
        3.  计算三重损失并反向传播更新参数（通常仅更新LoRA适配器）。
    *   **推理**: 
        1.  仅需输入 `<I0, Q>`。
        2.  像标准VLM一样直接生成答案文本。无需调用工具、无需解码潜在标记、无需使用[PRED]标记。

## 10. 总结评价
本文提出的PEARL框架是一项构思巧妙且实证扎实的工作。它精准地抓住了当前多模态潜在推理领域的关键痛点——特别是基于重建方法存在的训练-推理不匹配问题——并提出了一种简洁而有效的解决方案。通过将专家工具使用轨迹视为一个可预测的“视图”，并利用VLM自身的架构进行嵌入预测，该方法在概念上具有美感，在实践中也显示出强大的性能。
