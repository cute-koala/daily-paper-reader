## 摘要
常开式边缘摄像头产生连续的视频流，其中的冗余帧会将正确结果挤出前 k 个搜索结果，从而降低跨模态检索的性能。本文提出了一种流式检索架构：设备端 epsilon-net 过滤器仅保留语义新颖的帧，从而构建去噪的嵌入索引；跨模态适配器和云端重排序器则弥补了紧凑型编码器对齐能力弱的问题。在两个第一视角数据集（AEA、EPIC-KITCHENS）上，针对八个视觉语言模型（8M-632M）的测试表明，单次流式过滤器的表现优于离线替代方案（k-means、最远点采样、均匀采样、随机采样）。结合这些技术，该架构在使用 8M 设备端编码器的情况下，在留出数据上达到了 45.6% 的 Hit@5，且估计功耗仅为 2.7 mW。

## Abstract
Always-on edge cameras generate continuous video streams where redundant frames degrade cross-modal retrieval by crowding correct results out of top-k search. This paper presents a streaming retrieval architecture: an on-device epsilon-net filter retains only semantically novel frames, building a denoised embedding index; a cross-modal adapter and cloud re-ranker compensate for the compact encoder's weak alignment. A single-pass streaming filter outperforms offline alternatives (k-means, farthest-point, uniform, random) across eight vision-language models (8M-632M) on two egocentric datasets (AEA, EPIC-KITCHENS). Combined, the architecture reaches 45.6% Hit@5 on held-out data using an 8M on-device encoder at an estimated 2.7 mW.

*   **研究动机**：现有的紧凑型视觉语言模型（VLM）虽然能跑在边缘端，但其图文对齐能力较弱。作者发现，通过“去噪”（删除语义重复帧）构建的嵌入索引，反而能显著提升这些弱模型的检索性能。

### 2. 核心方法论
论文提出了一种四阶段的流式检索架构：
*   **设备端 $\epsilon$-net 过滤器（核心创新）**：采用单次遍历（Single-pass）算法。对于每一帧，计算其嵌入向量 $v_i$ 与已保留帧集合 $M$ 中所有向量的余弦相似度。如果最大相似度小于阈值 $\tau$，则保留该帧，否则丢弃。这在数学上构建了一个 $\epsilon$-net，保证了保留帧之间的语义差异性（分离性）和对原始流的覆盖性。
*   **跨模态适配器（LiT Adapter）**：在云端或配对手机上运行一个轻量级 MLP（525K 参数），将文本查询投影到图像嵌入空间，以弥补紧凑型编码器（如 TinyCLIP）的对齐缺陷。
*   **云端重排序器（Cloud Re-ranker）**：利用大型模型（如 SigLIP 2）对初步筛选出的候选帧进行二次打分，进一步提升精度。
*   **隐私保护设计**：设备仅传输 2KB 的嵌入向量而非原始图像，从设计上保护了视觉隐私。
