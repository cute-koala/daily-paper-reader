## 摘要
当前最具影响力的两种AI智能体认知架构框架——CoALA [21]与JEPA [12]，均缺乏一个具有自身持久性语义的显式知识层。这一缺失导致了一种范畴错误：系统对事实主张施加认知衰减，或以完全相同的更新机制处理事实与经验。我们调研了现有记忆系统中的持久性语义，识别出八个趋同点——从Karpathy的LLM知识库[10]到BEAM基准测试中近乎为零的矛盾消解得分[22]——所有这些都指向了相关的架构缺陷。我们提出了一个四层分解结构（知识、记忆、智慧、智力），其中每一层都具有根本不同的持久性语义：分别是无限取代、艾宾浩斯衰减、证据门控修订以及瞬时推理。配套的Python和Rust实现证明了这种架构分离是可行的。我们借用认知科学的术语作为一种有用的类比（知识与记忆的区分呼应了Tulving的三分法），但我们的分层是基于持久性语义需求而合理化的工程构造，而非基于神经架构。我们认为，这些区别要求在工程实现中采用不同的持久性语义，而目前尚无任何框架或系统提供这一点。

## Abstract
The two most influential cognitive architecture frameworks for AI agents, CoALA [21] and JEPA [12], both lack an explicit Knowledge layer with its own persistence semantics. This gap produces a category error: systems apply cognitive decay to factual claims, or treat facts and experiences with identical update mechanics. We survey persistence semantics across existing memory systems and identify eight convergence points, from Karpathy's LLM Knowledge Base [10] to the BEAM benchmark's near-zero contradiction-resolution scores [22], all pointing to related architectural gaps. We propose a four-layer decom position (Knowledge, Memory, Wisdom, Intelligence) where each layer has fundamentally different persistence semantics: indefinite supersession, Ebbinghaus decay, evidence-gated revision, and ephemeral inference respectively. Companion implementations in Python and Rust demonstrate the architectural separation is feasible. We borrow terminology from cognitive science as a useful analogy (the Knowledge/Memory distinction echoes Tulving's trichotomy), but our layers are engineering constructs justified by persistence-semantics requirements, not by neural architecture. We argue that these distinctions demand distinct persistence semantics in engineering implementations, and that no current framework or system provides this.

*   **背景与意义**：论文指出，两大主流认知架构框架——CoALA和JEPA——均未明确区分知识层与其他记忆类型。CoALA虽命名了“语义记忆”，但未赋予其与“情景记忆”不同的持久性语义；JEPA则完全缺失一个独立的、可持久化的知识层。这种架构缺陷在实践中导致了矛盾消解、时间推理等关键能力的低下（如BEAM基准测试中相关得分接近零）。本文旨在从工程角度解决这一根本性设计问题。

#### **2. 方法论：四层分解架构**
*   **核心思想**：提出一个基于**持久性语义差异**的四层认知基质分解模型。每一层对应不同类型的信息，并采用根本不同的存储、更新和检索机制。
    *   **知识层 (Knowledge)**：
        *   **内容**：“世界是什么样”——可验证的、有来源的事实。
        *   **持久性语义**：**无限期取代 (Indefinite Supersession)**。事实不会过期或衰减，只会被新证据所取代。旧事实被标记为“已取代”但仍保留以供溯源。
        *   **更新机制**：追加式存储 + 来源追踪。
        *   **作用域**：跨智能体共享。
    *   **记忆层 (Memory)**：
        *   **内容**：“发生了什么”——个人的、情境化的经验和观察。
        *   **持久性语义**：**艾宾浩斯衰减 (Ebbinghaus Decay)**。默认会随时间自然遗忘，除非通过重复访问得到巩固。
        *   **更新机制**：双时间事件溯源（记录系统时间和真实世界有效时间）。
        *   **作用域**：每个智能体私有。
    *   **智慧层 (Wisdom)**：
        *   **内容**：“什么方法有效”——从经验中提炼出的行为模式或规则（如“用户喜欢暗色模式”）。
        *   **持久性语义**: 持久的、基于证据的门控修订 **(Evidence-Gated Revision)**。规则的修改或提升需要达到特定的证据阈值（如多次独立会话的证实）。
        *   **更新机制**: 基于稳定性等级的审查与修订。
    *   **智力层 (Intelligence)**：
        - 内容: “推理、规划和行动的能力”。
        - 持久性语义: 瞬时性的 **(Ephemeral Inference)**。仅在推理时存在，不直接留下痕迹；其效果通过影响其他三层来体现。

*   **关键技术细节/原则**
    1.  `确定性操作`：配套实现的核心操作（衰减计算、图遍历等）均为确定性算法，无LLM调用或网络依赖，保证了可预测性和离线能力。
    2.  `消费者特质抽象`：将嵌入生成、摘要生成等功能抽象为接口 (`Consumer Traits`)，由调用方提供具体实现，使内存引擎本身轻量且模型无关。
    3.  `显式路由`：查询需要被正确路由到相应的层次进行处理。
