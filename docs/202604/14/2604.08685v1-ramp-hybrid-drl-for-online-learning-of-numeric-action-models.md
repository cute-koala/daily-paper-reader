## 摘要
自动规划算法需要一个指定每个动作前提条件和效果的动作模型，但获取此类模型通常很困难。从观察中学习动作模型是可行的，但现有的数值领域算法是离线的，需要专家轨迹作为输入。我们提出了强化学习、动作模型学习和规划（RAMP）策略，用于通过与环境的交互在线学习数值规划动作模型。RAMP同时训练深度强化学习（DRL）策略，从过去的交互中学习数值动作模型，并在可能时利用该模型规划未来动作。这些组件形成了一个正向反馈循环：RL策略收集数据以改进动作模型，而规划器则生成计划以继续训练RL策略。为了促进RL与数值规划的集成，我们开发了Numeric PDDLGym，这是一个将数值规划问题转换为Gym环境的自动化框架。在标准IPC数值领域的实验结果表明，在可解性和计划质量方面，RAMP显著优于著名的DRL算法PPO。

## Abstract
Automated planning algorithms require an action model specifying the preconditions and effects of each action, but obtaining such a model is often hard. Learning action models from observations is feasible, but existing algorithms for numeric domains are offline, requiring expert traces as input. We propose the Reinforcement learning, Action Model learning, and Planning (RAMP) strategy for learning numeric planning action models online via interactions with the environment. RAMP simultaneously trains a Deep Reinforcement Learning (DRL) policy, learns a numeric action model from past interactions, and uses that model to plan future actions when possible. These components form a positive feedback loop: the RL policy gathers data to refine the action model, while the planner generates plans to continue training the RL policy. To facilitate this integration of RL and numeric planning, we developed Numeric PDDLGym, an automated framework for converting numeric planning problems to Gym environments. Experimental results on standard IPC numeric domains show that RAMP significantly outperforms PPO, a well-known DRL algorithm, in terms of solvability and plan quality.

# RAMP 论文深度分析总结

## 1. 核心问题与整体含义
*   **研究背景**：自动化规划（Automated Planning）是解决序列决策问题的可靠方法，但其核心依赖一个精确的动作模型（Action Model），即定义每个动作的前提条件和效果。手动构建此类模型，尤其是在涉及离散和连续状态变量的**数值规划（Numeric Planning）** 领域，非常困难。
*   **现有局限**：虽然存在从观察数据中学习动作模型的方法，但针对数值领域的现有算法大多是**离线（offline）** 的，需要预先提供专家执行轨迹作为输入。这限制了其在需要通过与未知环境交互来学习的场景中的应用。
*   **研究动机与目标**：本文旨在解决**在线学习数值动作模型**的问题。作者提出了一种名为RAMP的混合策略，它无需专家轨迹，而是通过与环境的直接交互来同时学习策略和动作模型。其整体含义在于弥合了深度强化学习（DRL）在在线探索方面的优势与符号规划在高效推理和泛化能力之间的鸿沟。

## 2. 方法论
*   **核心思想 (RAMP策略)**：RAMP是一个集成了三个组件的循环框架：
    1.  **深度强化学习 (DRL)**：使用PPO等算法训练一个策略，用于在环境中进行目标导向的探索和数据收集。
    2.  **动作模型学习 (AML)**：利用DRL收集到的交互轨迹（状态-动作序列），通过离线AML算法（本文选用NSAM）迭代地学习和精炼一个数值规划动作模型。
    3.  **规划器 (Planner)**：当学习到的动作模型足够好时，使用规划器（如Metric-FF）基于该模型生成高效的计划。
    *   **正向反馈循环**：这三个组件形成闭环——RL探索为AML提供数据以改进模型；改进后的模型使规划器能生成更高质量的计划；这些计划又为RL提供了高质量的训练数据，加速其收敛并提升性能。
*   **关键技术细节**：
    *   **安全保证 (Safety Guarantee)**：选用的NSAM算法能保证学到的动作模型是“安全”的——任何在该模型中有效的计划，在真实未知环境中也是可执行的并能达到目标。这确保了基于模型的规划的可靠性。
    *   **无效动作屏蔽 (Invalid Action Masking)**：
        *   为了解决PPO等DRL算法在遵循规划器生成的“专家”行动时可能出现的梯度更新失效问题（因策略概率差异过大），作者采用了屏蔽机制。在执行计划时，只将计划中的行动视为有效行动进行梯度更新。
        *   同时屏蔽之前观察到在当前状态下不可行的行动，防止智能体重复尝试无效操作。
    *   **Numeric PDDLGym框架**：为了将数值PDDL规划问题无缝接入标准DRL训练流程（如RLlib），作者开发了此框架。它能自动将PDDL2.1描述的领域和问题转换为Gym环境：
        *   将符号化的状态（谓词、函数）扁平化为固定长度的数值观测向量。
        *   将所有提升的动作实例化为离散的动作空间。
        *   定义奖励函数（到达目标获得+1奖励）。
        *   处理前提条件不满足的情况（保持状态不变）。

## 6. 实验结果与发现
*   **主要性能对比 (RAMP vs. PPO)**：
    1.  **成功率**：在所有测试领域中，RAMP在收敛速度和最终成功率上都显著优于纯PPO基线。特别是在“Large”规模问题上，PPO往往难以学习或完全失败，而RAMP通过引入规划能力，能够稳定地达到高成功率。
    2.  **解长度**：RAMP生成的解决方案（无论是通过规划器还是RL策略执行）通常比PPO找到的解决方案更短、更高效。这表明学到的动作模型和规划器的介入有助于找到更优的行动序列。
*   **关键洞察**：
    1.  **正向循环的有效性**：实验验证了DRL探索、AML学习和符号规划三者之间正向反馈循环的有效性。随着训练进行，学到的动作模型质量（Precision/Recall）逐步提升，规划器被调用的频率和成功率也随之增加。
    2.  **安全保证的价值**：NSAM提供的“安全”模型保证了基于模型的规划不会导致灾难性失败（如执行无效动作），这是RAMP能够稳定运行的关键。
    3.  **从探索到利用的过渡**：在训练初期，智能体主要依赖DRL进行随机探索以收集数据。随着动作模型的完善，智能体越来越多地依赖规划器生成精确的计划来完成任务，实现了从无模型探索到基于模型利用的自然过渡。
    4.  **对复杂任务的泛化能力**：在最具挑战性的Pogo Stick域中，RAMP展现出了处理复杂资源管理和多步合成任务的能力，而纯DRL方法则难以应对。

## 7. 结论与贡献
*   **核心结论**：本文提出的RAMP框架成功地证明了在线学习数值动作模型的可行性。它通过结合深度强化学习的探索能力和符号规划的推理泛化能力，克服了纯无模型RL在样本效率和解质量上的不足，以及纯符号方法对预定义模型的依赖。
*   **主要贡献**：
    1.  **方法论贡献**：提出了首个用于在线学习数值动作模型的混合DRL框架（RAMP），并设计了有效的组件集成机制（如无效动作屏蔽）。
    2.  **技术贡献**：开发了Numeric PDDLGym框架，为数值规划问题与标准DRL库的集成提供了便利工具。
    3.  **实证贡献**：在多个标准及自定义基准上进行了全面实验，验证了RAMP相对于强基线的优越性能、鲁棒性和可扩展性。

## 8. 局限性与未来工作
*   **局限性**：
    1.  **状态表示瓶颈**：当前将符号状态扁平化为固定长度向量的方法可能面临维度灾难问题，且可能丢失部分结构信息。对于状态空间极大或高度结构化的领域可能不适用。
    2.  **离散化动作空间限制**：将提升的动作实例化为离散集合的方法限制了其处理连续或参数化动作的能力。
    3.  **AML算法的依赖性**：RAMP的性能部分依赖于底层AML算法（NSAM）的质量和效率。对于更复杂的数值关系或存在噪声的数据，可能需要更强大的学习算法。
    4.  **计算开销权衡**：虽然最终计划高效，但训练过程中交替进行RL、AML和规划增加了计算复杂度。在线规划和模型更新的频率需要仔细权衡。
*   **未来方向建议**：
    1.  探索使用图神经网络等更高级的表示学习方法来处理结构化状态和动作。
    2.  研究如何将框架扩展至连续动作空间或参数化动作的学习。
    3.  集成更鲁棒的动作模型学习算法以处理噪声或部分可观察的环境。
    4.  优化整个系统的计算效率，例如研究更智能的触发机制来决定何时调用规划器或更新模型。
