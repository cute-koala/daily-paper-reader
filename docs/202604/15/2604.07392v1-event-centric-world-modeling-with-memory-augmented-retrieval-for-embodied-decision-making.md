## 摘要
在动态且安全关键的环境中运行的自主智能体，需要兼具计算高效性和物理基础性的决策框架。然而，现有许多方法依赖于端到端学习，这通常缺乏可解释性，也缺少确保与物理约束保持一致的显式机制。为此，我们提出了一种用于具身决策的、基于记忆增强检索的事件中心化世界建模框架。该框架将环境表示为一组结构化的语义事件，并将其编码为排列不变的潜在表示。决策通过对先前经验知识库的检索来执行，其中每个条目将一个事件表示与相应的操控动作相关联。最终动作通过检索到的解决方案的加权组合计算得出，从而在决策与存储的经验之间建立了透明的联系。所提出的设计实现了对动态环境的结构化抽象，并通过基于案例的推理支持可解释的决策。此外，将融入物理知识的先验信息整合到检索过程中，有助于选择与观测到的系统动力学相一致的操控动作。在无人机飞行场景中的实验评估表明，该框架能够在实时控制约束下运行，同时保持行为的可解释性和一致性。

## Abstract
Autonomous agents operating in dynamic and safety-critical environments require decision-making frameworks that are both computationally efficient and physically grounded. However, many existing approaches rely on end-to-end learning, which often lacks interpretability and explicit mechanisms for ensuring consistency with physical constraints. In this work, we propose an event-centric world modeling framework with memory-augmented retrieval for embodied decision-making. The framework represents the environment as a structured set of semantic events, which are encoded into a permutation-invariant latent representation. Decision-making is performed via retrieval over a knowledge bank of prior experiences, where each entry associates an event representation with a corresponding maneuver. The final action is computed as a weighted combination of retrieved solutions, providing a transparent link between decision and stored experiences. The proposed design enables structured abstraction of dynamic environments and supports interpretable decision-making through case-based reasoning. In addition, incorporating physics-informed knowledge into the retrieval process encourages the selection of maneuvers that are consistent with observed system dynamics. Experimental evaluation in UAV flight scenarios demonstrates that the framework operates within real-time control constraints while maintaining interpretable and consistent behavior.

## 2. 方法论
*   **核心思想**：构建一个“事件-检索-行动”（ERA）框架。智能体将当前环境感知编码为一个“事件列表”，并将其压缩为“事件代码”作为查询向量。通过在一个存储了历史（事件代码，操控动作）对的“知识库”中进行相似性检索，加权组合最相关的历史动作来生成当前决策。
*   **关键技术细节**：
    *   **事件中心化表示 (Event-Centric Representation)**：
        *   环境被建模为一个结构化的事件列表 `Et`，包含检测到的物体ID、位置、运动状态、环境状态、自身状态和目标上下文等信息。
        *   使用置换不变编码器（如DeepSets或Transformer）将 `Et` 映射为紧凑的潜在表示 `zt`（事件代码）。
    *   **记忆增强检索 (Memory-Augmented Retrieval)**：
        *   维护一个知识库 `M = {(zi, ai, ri)}`，存储历史经验（事件代码、对应动作、奖励/可行性分数）。
        *   使用近似最近邻搜索（ANN, 如FAISS）根据相似度 `sim(zt, zi)` 从 `M` 中检索Top-k个相关经验。
        *   最终动作 `at` 是检索到的动作 `ai` 的加权和：`at = Σ wi * ai`，权重 `wi` 由相似度经softmax归一化得到。
    *   **物理知识注入与稳定性保证 (Physics-Informed & Stability Guarantees)**：
        *   **物理一致性正则器 (R_phys)**：在训练目标中引入项 `Σ wi * d_phys(zt, zi)`，惩罚与当前物理状态不匹配的动作选择。
        *   **收缩性潜在动力学 (Contractive Latent Dynamics)**：约束潜在状态的线性转移算子 Ψ (`zt+1 = Ψ zt + Γ at + εt`)满足谱半径 ρ(Ψ) < 1，确保长期预测有界且稳定（类似Lyapunov稳定性）。
        *   **聚类贝叶斯选择 (Clustered Bayesian Selection)**：为避免对方向相反的安全动作进行平均导致危险动作（“平均至碰撞”问题），先将检索到的动作按方向聚类，然后选择总权重最高的簇进行内部加权平均。
*   **算法流程**：
    1.  **预训练阶段 (Algorithm 1)**：利用专家演示或仿真数据初始化知识库。通过优化两个损失函数来训练编码器和知识库嵌入：(a)度量学习损失 `L_metric`，使潜在空间距离反映物理状态差异；(b)模仿学习损失 `L_imitation`。
    2.  **在线决策与训练阶段 (Algorithm 2)**：
        a.  从传感器输入提取特征并编码为事件列表 `Et`。
        b.  计算潜在事件代码 `zt = f(Et)`。
        c.  从知识库 `M`中检索最相似的k个经验对 `(zi, ai)`。
        d.  计算权重并生成最终动作 `at = Σ wi*ai`。
        e.  执行动作并观察结果，生成状态码存入经验缓冲区用于后续离线策略优化。
