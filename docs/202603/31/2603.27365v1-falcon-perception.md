## 摘要
以感知为中心的系统通常采用模块化的编码器-解码器流水线实现：用于特征提取的视觉骨干网络和用于任务预测的独立解码器（或后期融合模块）。这提出了一个核心问题：这种架构分离是必不可少的，还是单个早期融合堆栈就可以在大规模上同时完成感知和任务建模？我们介绍了 Falcon Perception，这是一个统一的密集 Transformer，它从第一层开始就在共享参数空间中处理图像块和文本标记，并使用混合注意力模式（图像标记之间为双向，预测标记为因果）将全局视觉上下文与自回归、变长实例生成相结合。为了保持密集输出的实用性，Falcon Perception 保留了轻量级的标记接口，并使用专门的头部解码连续空间输出，从而实现并行的分辨率掩码预测。我们的设计崇尚简洁：我们保留了单个可扩展的骨干网络，并将复杂性转向数据和训练信号，仅在输出连续且密集的地方添加小型头部。在 SA-Co 上，Falcon Perception 将掩码质量提高到 68.0 Macro-F$_1$，而 SAM3 为 62.3。我们还引入了 PBench，这是一个针对组合提示（OCR、空间约束、关系）和密集长上下文场景的基准测试，模型在这些场景中表现出更好的增益。最后，我们将同样的早期融合方案扩展到 Falcon OCR：这是一个参数量仅为 300M 的紧凑模型，在 olmOCR 上达到 80.3%，在 OmniDocBench 上达到 88.64。

## Abstract
Perception-centric systems are typically implemented with a modular encoder-decoder pipeline: a vision backbone for feature extraction and a separate decoder (or late-fusion module) for task prediction. This raises a central question: is this architectural separation essential or can a single early-fusion stack do both perception and task modeling at scale? We introduce Falcon Perception, a unified dense Transformer that processes image patches and text tokens in a shared parameter space from the first layer, using a hybrid attention pattern (bidirectional among image tokens, causal for prediction tokens) to combine global visual context with autoregressive, variable-length instance generation. To keep dense outputs practical, Falcon Perception retains a lightweight token interface and decodes continuous spatial outputs with specialized heads, enabling parallel high-resolution mask prediction.   Our design promotes simplicity: we keep a single scalable backbone and shift complexity toward data and training signals, adding only small heads where outputs are continuous and dense. On SA-Co, Falcon Perception improves mask quality to 68.0 Macro-F$_1$ compared to 62.3 of SAM3. We also introduce PBench, a benchmark targeting compositional prompts (OCR, spatial constraints, relations) and dense long-context regimes, where the model shows better gains. Finally, we extend the same early-fusion recipe to Falcon OCR: a compact 300M-parameter model which attains 80.3% on olmOCR and 88.64 on OmniDocBench.

### 1. 核心问题与背景
*   **核心问题**：传统的视觉感知系统（如开集分割、OCR）通常采用“视觉骨干网络（编码器）+ 任务解码器”的模块化架构。论文探讨这种架构分离是否必要，以及是否可以用一个**单一的、早期融合（Early-fusion）的密集 Transformer 堆栈**同时完成视觉特征提取和复杂的感知任务建模。
*   **研究动机**：模块化设计虽然有效，但增加了任务特定的复杂性（如模态融合、查询匹配、后处理），限制了视觉与文本特征在深层的交互。Falcon Perception 旨在通过简化架构，将复杂度从模型结构转向数据和训练信号。

### 2. 方法论
*   **核心思想**：将图像补丁（Patches）和文本标记（Tokens）视为统一序列，从第一层起就在共享参数空间内处理。
*   **关键技术细节**：
    *   **混合注意力掩码（Hybrid Attention）**：图像 Token 之间采用双向注意力（类似编码器），而文本和任务 Token 对图像全可见但对自己及后续 Token 采用因果掩码（类似解码器）。
    *   **感知链（Chain-of-Perception）**：将实例分割分解为有序序列：`<coord>（中心坐标）→ <size>（尺寸）→ <seg>（分割掩码）`。这种粗到精的策略稳定了训练。
    *   **专用轻量化头**：
        *   **坐标与尺寸**：使用傅里叶特征（Fourier Features）映射连续坐标，解决神经网络对高频空间细节学习难的问题。
        *   **分割掩码**：通过 `<seg>` Token 的隐藏状态与上采样后的图像特征图进行点积，实现并行的高分辨率掩码生成。
    *   **3D 旋转位置嵌入（GGRoPE）**：引入 Golden Gate RoPE 处理 2D 空间位置，增强模型对不同长宽比和旋转的鲁棒性。
*   **训练流程**：
    1.  **多教师蒸馏**：从 DINOv3 和 SigLIP2 初始化权重。
    2.  **感知训练**：在 685 GT（十亿标记）数据上进行三阶段训练（上下文列表、任务对齐、长上下文微调）。

### 5. 实验数量与充分性

*   **消融实验**：非常充分。针对优化器（Muon vs AdamW）、特征正则化（Gram Loss）、注意力掩码策略、实例排序方式（随机 vs 尺寸 vs 栅格扫描）、分辨率缩放等关键设计点均做了定量对比。

### 6. 主要结论与发现
*   **性能领先**：在 SA-Co 上，Falcon Perception 的掩码质量（68.0 Macro-F1）显著优于 SAM 3（62.3）。
*   **组合能力强**：在 PBench 的空间关系（L3）和复杂关系（L4）任务中，0.6B 的模型表现甚至超过了 30B 的 Qwen3-VL。
*   **OCR 优势**：300M 的 Falcon OCR 在 olmOCR 上达到 80.3%，证明了早期融合架构在处理文本密集型视觉任务时的极高效率。
*   **分辨率敏感性**：发现分辨率对“检测是否存在”影响较小，但对“精确掩码生成”存在明显的相位偏移（Phase Transition），高分辨率是密集感知的先决条件。

### 7. 优点
*   **架构极简**：取消了复杂的编码器-解码器分离，实现了真正的单骨干网络感知。
*   **推理高效**：通过自回归生成少量 Token 配合并行掩码头，解决了纯自回归模型处理密集输出时的速度瓶颈。
*   **泛化性好**：在 OCR、空间推理和拥挤场景下表现出极强的鲁棒性。
*   **采样潜力**：展示了自回归模型通过 Pass@k 采样可以显著提升性能，暗示了未来结合强化学习（RL）的巨大空间。

### 8. 不足与局限
*   **存在性校准（Presence Calibration）**：在处理“负样本”（即图中不存在提示对象）时，MCC 指标略逊于 SAM 3，这源于自回归模型天然缺乏“空类别”机制，需依赖更强的负采样训练。
*   **训练成本**：相比于全并行的 DETR 类模型，单堆栈自回归模型的训练和长序列处理开销相对较高。
*   **OCR 细节**：在处理极小文字（TinyText）和严重降级的历史扫描件（OldScan）时，受限于 300M 的参数量，性能仍有提升空间。
