## 摘要
大语言视觉模型 (LVLMs) 通过利用大语言模型 (LLMs) 的推理能力，在各种多模态任务中表现出强大的性能。然而，处理视觉复杂且信息丰富的图像（如信息图表或文档布局）需要这些模型生成大量的视觉 token，从而导致显著的计算开销。为了解决这一问题，我们提出了 PinPoint，这是一个新颖的两阶段框架，它首先识别与指令相关的图像区域，然后对其进行细化以提取细粒度的视觉特征，从而提高推理能力和效率。我们方法的核心是指令-区域对齐 (Instruction-Region Alignment)，它利用视觉输入和文本指令来定位相关区域。我们进一步引入了新的标注，为挑战性的 VQA 基准测试（InfographicVQA、MultiPageDocVQA 和 SinglePageDocVQA）中的指令相关区域提供更丰富的真值 (ground-truth) 监督。实验结果表明，与现有方法相比，PinPoint 不仅实现了卓越的准确性，还通过最小化无关的视觉 token 减少了计算开销。

## Abstract
Large Vision-Language Models (LVLMs) have shown strong performance across various multimodal tasks by leveraging the reasoning capabilities of Large Language Models (LLMs). However, processing visually complex and information-rich images, such as infographics or document layouts, requires these models to generate a large number of visual tokens, leading to significant computational overhead. To address this, we propose PinPoint, a novel two-stage framework that first identifies instruction-relevant image regions and then refines them to extract fine-grained visual features for improved reasoning and efficiency. Central to our approach is the Instruction-Region Alignment, which localizes relevant regions using both visual input and textual instructions. We further introduce new annotations that provide richer ground-truth supervision for instruction-relevant regions across challenging VQA benchmarks: InfographicVQA, MultiPageDocVQA, and SinglePageDocVQA. Experimental results show that PinPoint not only achieves superior accuracy compared to existing methods but also reduces computational overhead by minimizing irrelevant visual tokens.

### 论文总结：Focus, Don't Prune (PinPoint)

#### 1. 核心问题与整体含义（研究动机和背景）
大型视觉语言模型（LVLM）在处理信息密集型图像（如信息图表、复杂文档、网页截图）时，通常需要极高的分辨率来捕捉细微的文字和布局信息。这导致生成的视觉 Token 数量激增，带来了巨大的计算开销。
现有的**Token 剪枝（Pruning）**方法存在三大缺陷：
*   **注意力图不可靠**：基于 LLM 注意力权重的剪枝容易导致幻觉。
*   **语义碎片化**：在 Token 级别进行剪枝会破坏文字或目标的连续语义。
*   **上下文纠缠**：全局自注意力机制使得相关区域的特征被无关背景噪声污染。
**研究动机**：受人类“先全局扫描、再局部聚焦”的视觉策略启发，作者提出 **PinPoint** 框架，主张“聚焦（Focus）”而非简单的“剪枝（Prune）”，通过识别并精炼指令相关区域来提升效率与精度。

#### 2. 方法论：核心思想与技术细节
PinPoint 是一个两阶段框架，旨在从海量视觉 Token 中提取最关键的细粒度特征：

*   **第一阶段：区域选择 (Region Selection)**
    *   **特征提取**：使用滑动窗口（Sliding Window）在原始视觉特征图上移动，将窗口内的 Token 聚合为区域级表示。
    *   **指令-区域对齐 (Instruction-Region Alignment)**：引入一组可学习的引导查询（Guidance Queries），通过注意力机制分别与视觉区域特征和文本指令特征交互，将两者映射到统一空间。
    *   **自适应选择**：计算区域特征与指令特征的余弦相似度，按得分排序，自适应选择覆盖图像比例为 $r$（如 60%）的 Top-k 区域。
*   **第二阶段：区域精炼 (Region Refinement)**
    *   **局部重编码**：仅对选定的相关区域进行高分辨率重新编码。这一步能剥离无关的全局背景噪声，提取更纯净、更细致的视觉特征。
    *   **最终推理**：将精炼后的少量视觉 Token 与文本指令一同输入 LLM 生成答案。
*   **损失函数**：采用双重对比学习策略：
    1.  **模态间对比损失 ($L_{inter}$)**：对齐视觉区域与对应的文本指令。
    2.  **图像内对比损失 ($L_{intra}$)**：在同一张图中，拉近指令与相关区域的距离，推开指令与无关区域的距离。

*   **数据集**：

    *   **泛化性/幻觉测试**：MMMU, MMMU-Pro, TextVQA, POPE, CHAIR, AMBER。

*   **对比方法**：

    *   **基准**：Vanilla（全量 Token 模型）和随机采样。
*   **新标注**：作者为 InfoVQA 等数据集提供了更丰富的标注，不仅包含答案框，还包含支持推理的“证据框（Evidence Bboxes）”。

*   **硬件**：使用 2 张 **NVIDIA A100 GPU**。
*   **训练效率**：

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 4 个核心 VQA 任务、3 个幻觉评估基准以及 3 个跨领域泛化测试，实验设计非常全面。
*   **消融实验**：针对图像内对比损失 ($L_{intra}$)、区域精炼模块、区域覆盖阈值 $r$ 以及标注类型（仅答案框 vs. 包含证据框）均做了详细对比。
*   **客观性**：在相同的 LLM 底座和计算预算下对比了多种 SOTA 剪枝算法，证明了 PinPoint 在相同 Token 预算下具有更高的 ANLS（平均归一化编辑距离）得分。

#### 6. 主要结论与发现
*   **精度超越全量模型**：在 InfoVQA 等任务中，PinPoint 仅使用约 65% 的计算量，准确率却比处理全量 Token 的 Vanilla 模型更高（InfoVQA 提升了 18.5%）。
*   **缓解幻觉**：通过精准定位相关区域并去除噪声，模型在 POPE 和 CHAIR 等幻觉指标上表现更优，生成的描述更具事实性。
*   **强泛化性**：仅在 GQA 上训练的模块，在 MMMU 和 TextVQA 等未见数据集上依然表现出极强的性能提升。

#### 7. 优点（亮点）
*   **策略转变**：从“被动剪枝”转向“主动聚焦”，解决了剪枝方法破坏语义连续性的固有问题。
*   **高质量数据贡献**：提出了一个自动化的标注管线，并发布了包含“推理证据区域”的新数据集，这对社区研究视觉定位推理极具价值。
*   **极高的训练效率**：无需微调 LLM 或视觉编码器，仅训练轻量级投影层，适合资源受限的场景。

#### 8. 不足与局限
*   **第一阶段依赖性**：如果第一阶段的区域选择出现偏差（未覆盖到关键信息），第二阶段的精炼也无法挽回错误。
*   **滑动窗口限制**：固定的滑动窗口尺寸可能无法完美匹配所有形状的目标，虽然有 Top-k 聚合，但仍存在边界切割风险。
*   **实时性挑战**：虽然 FLOPs 降低了，但两阶段推理（需要两次视觉编码过程，尽管第二次是局部的）在某些硬件架构上可能会带来额外的调度延迟。
