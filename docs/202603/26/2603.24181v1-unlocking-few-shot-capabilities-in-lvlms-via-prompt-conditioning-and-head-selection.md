## 摘要
当前的大语言视觉模型 (LVLM) 在图像描述、视觉问答和 OCR 等许多零样本任务中表现出色。然而，这些模型在图像分类任务中的表现较差，逊于基于 CLIP 的方法。值得注意的是，这一差距令人惊讶，因为许多 LVLM 使用了经过 CLIP 预训练的视觉编码器。然而，LVLM 本质上并不受限于 CLIP 那种具有独立视觉和文本编码器的架构。在 CLIP 中，这种分离使得分类偏向于类名匹配，而非联合视觉-文本推理。在本文中，我们展示了尽管 LVLM 的原始性能不佳，但它们可以在推理过程中利用提示词调节来提高视觉特征的类别可分性，且 LVLM 的内部表示（尤其是注意力头）在零样本和少样本分类中的表现可以超越模型本身。我们引入了注意力头集成分类器 (HEC)，以弥合基于 CLIP 和基于 LVLM 的分类方法之间的性能差距。受高斯判别分析的启发，HEC 对最具辨别力的视觉和文本头进行排序，并将它们组合成一个无需训练的分类器。我们证明了 HEC 在 12 个数据集的少样本和零样本分类中均达到了最先进的性能。

## Abstract
Current Large Vision Language Models (LVLMs) excel at many zero-shot tasks like image captioning, visual question answering and OCR. However, these same models suffer from poor performance at image classification tasks, underperforming against CLIP-based methods. Notably, this gap is surprising because many LVLMs use CLIP-pretrained vision encoders. Yet LVLMs are not inherently limited by CLIP's architecture with independent vision and text encoders. In CLIP, this separation biases classification toward class-name matching rather than joint visual-text reasoning. In this paper we show that, despite their poor raw performance, LVLMs can improve visual feature class separability at inference using prompt conditioning, and LVLMs' internal representations, especially attention heads, can outperform the model itself at zero-shot and few-shot classification. We introduce Head Ensemble Classifiers (HEC) to bridge the performance gap between CLIP-based and LVLM-based classification methods. Inspired by Gaussian Discriminant Analysis, HEC ranks the most discriminative vision and text heads and combines them into a training-free classifier. We show that HEC achieves state-of-the-art performance in few-shot and zero-shot classification across 12 datasets.

*   **核心问题**：尽管大型视觉语言模型（LVLM）在多模态任务中表现卓越，但在基础的**图像分类**任务上，其性能往往显著低于基于 CLIP 的传统方法，甚至低于其自身内置的 CLIP 视觉编码器。
*   **研究动机**：作者发现 LVLM 的最终输出（Summary Token）在分类时存在信息损失。由于 LVLM 采用联合视觉-文本推理架构，其内部表征（尤其是注意力头）其实蕴含了比最终输出更丰富的判别性特征。研究旨在通过无需微调的方法，挖掘并组合这些内部“专家头”，以弥合 LVLM 与 CLIP 之间的分类性能差距。

### 2. 方法论：HEC (Head Ensemble Classifiers)
核心思想是利用**提示词调节（Prompt Conditioning）**引导模型表征，并结合**高斯判别分析（GDA）**筛选出表现最优的注意力头进行集成。

*   **提示词调节**：提出三个级别的引导：
    *   **任务调节**：如“这张图片里有什么？”
    *   **领域调节**：针对细粒度任务，如“这只狗是什么品种？”
    *   **类别调节**：在提示词中加入候选类别列表。
*   **注意力头筛选与集成**：
    *   **HEC-V（视觉少样本分类器）**：在给定少量标注样本（支持集）的情况下，利用 GDA 假设每个头的特征符合高斯分布，计算各头在支持集上的类可分性得分，选取 Top-k 个头。
    *   **HEC-T（文本零样本分类器）**：利用类名生成的文本特征，筛选出在支持集上零样本准确率最高的头。
    *   **HEC-VT（混合分类器）**：将 HEC-V 和 HEC-T 的预测概率进行加权融合。
*   **算法流程**：输入图像和提示词 -> 提取所有层的注意力头特征 -> 基于支持集进行 GDA 评分排序 -> 集成 Top-k 头的概率输出 -> 得到最终分类结果。

### 6. 主要结论与发现
*   **提示词的魔力**：提示词调节能显著提高 LVLM 内部特征的类别可分性，这是传统视觉模型无法做到的。
*   **内部表征更优**：LVLM 内部约 25% 的注意力头在分类任务上的表现优于模型的最终输出。
*   **性能突破**：HEC 在少样本和零样本任务中均达到了 SOTA，成功弥合了 LVLM 与 CLIP 之间的差距，甚至在非物体中心的数据集（如 DTD 纹理）上超越了 CLIP。

### 7. 优点
*   **无需训练**：作为一种插件式方法，不需要昂贵的显存进行参数更新。
*   **高效性**：利用闭式解（Closed-form）进行头筛选，计算开销极低。
*   **灵活性**：通过简单的提示词即可实现领域自适应，无需针对特定领域重新训练模型。
*   **可解释性**：通过注意力图可视化证明了筛选出的“专家头”确实关注到了领域相关的关键特征。

### 8. 不足与局限
*   **API 访问受限**：该方法需要提取模型中间层的注意力头特征，对于仅提供 API 访问（如 GPT-4V）的闭源模型无法直接使用。
*   **推理开销**：虽然分类器构建很快，但 LVLM 本身的推理成本（FLOPs）远高于轻量级的 CLIP 模型。
*   **类别规模限制**：类别调节（Class Conditioning）受限于 LLM 的上下文窗口长度，当类别数量极多（如上千类）时，性能会下降。
*   **特征维度**：需要存储和处理中间层特征，对内存带宽有一定要求。
