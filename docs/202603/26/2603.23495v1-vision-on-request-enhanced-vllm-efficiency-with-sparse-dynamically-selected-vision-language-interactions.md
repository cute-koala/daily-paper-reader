## 摘要
现有的提高大型视觉语言模型 (LVLM) 效率的方法主要基于视觉 token 缩减的概念。然而，这种方法会产生信息瓶颈，从而损害性能，特别是在需要细粒度理解和推理的挑战性任务上。在这项工作中，我们通过引入 VISion On Request (VISOR) 来挑战这一范式，这是一种在不丢弃视觉信息的情况下降低推理成本的方法。VISOR 并非压缩图像，而是通过稀疏化图像和文本 token 之间的交互来提高效率。具体而言，语言模型通过一组少量且经过策略性布置的注意力层来关注全套高分辨率视觉 token：通用的视觉上下文由文本-图像之间的高效交叉注意力提供，而少数位置合理且动态选择的自注意力层则负责细化视觉表示本身，从而在需要时实现复杂的、高分辨率的推理。基于这一原则，我们首先通过改变自注意力层的数量，在不同的计算预算范围内训练一个单一的通用网络，然后引入一种轻量级的策略机制，根据每个样本的复杂度动态分配视觉计算。大量的实验表明，VISOR 在各种基准测试中大幅降低了计算成本，同时达到或超过了最先进的结果，并且在需要详细视觉理解的挑战性任务中表现出色。

## Abstract
Existing approaches for improving the efficiency of Large Vision-Language Models (LVLMs) are largely based on the concept of visual token reduction. This approach, however, creates an information bottleneck that impairs performance, especially on challenging tasks that require fine-grained understanding and reasoning. In this work, we challenge this paradigm by introducing VISion On Request (VISOR), a method that reduces inference cost without discarding visual information. Instead of compressing the image, VISOR improves efficiency by sparsifying the interaction between image and text tokens. Specifically, the language model attends to the full set of high-resolution visual tokens through a small, strategically placed set of attention layers: general visual context is provided by efficient cross-attention between text-image, while a few well-placed and dynamically selected self-attention layers refine the visual representations themselves, enabling complex, high-resolution reasoning when needed. Based on this principle, we first train a single universal network on a range of computational budgets by varying the number of self-attention layers, and then introduce a lightweight policy mechanism that dynamically allocates visual computation based on per-sample complexity. Extensive experiments show that VISOR drastically reduces computational cost while matching or exceeding state-of-the-art results across a diverse suite of benchmarks, and excels in challenging tasks that require detailed visual understanding.

### VISion On Request (VISOR) 论文深度总结

#### 1. 核心问题与整体含义（研究动机和背景）
大型视觉语言模型（LVLM）在处理高分辨率图像时，由于视觉标记（Token）数量庞大，推理成本极高。现有的主流优化方案主要采用**Token 缩减或压缩**（如剪枝、合并），但这会造成不可逆的信息丢失，导致模型在需要细粒度理解（如 OCR、文档分析、复杂推理）的“硬任务”上性能大幅下降。
本文提出了 **VISOR (VISion On Request)**，其核心动机是：**与其减少视觉 Token 的数量，不如稀疏化视觉与语言之间的计算交互**。研究发现，模型并非在每一层都需要深度处理图像，通过动态选择交互层，可以在不丢弃任何视觉细节的前提下显著提升效率。

#### 2. 论文提出的方法论
VISOR 改变了传统 LVLM 每层都进行全量图文交互的模式，采用以下核心技术：
*   **解耦交互机制**：将 LLM 层分为三类：
    1.  **纯文本层**：仅处理文本 Token，不涉及视觉计算。
    2.  **交叉注意力层 (Cross-Attention, CA)**：文本 Token 作为 Query 去查询视觉 Token。这种方式只提取视觉信息而不更新视觉表示，计算开销极低。
    3.  **自注意力层 (Self-Attention, SA)**：执行完整的图文混合自注意力，用于更新和精炼视觉表示，支持复杂的层次化推理。
*   **通用模型训练 (Universal Training)**：训练一个单一网络，使其在训练过程中随机采样不同的 SA 层配置（从 0 到最大层数），从而使模型能够稳健地适配各种计算预算。
*   **动态自适应推理 (Adaptive Inference)**：引入一个轻量级的**策略网络（Router）**。该网络根据输入样本的复杂度，在推理时动态决定开启哪些 SA 层。例如，简单任务（如识别颜色）可能只需 CA 层，而复杂任务（如阅读文档）则调用更多 SA 层。
*   **位置编码优化**：采用一维深度卷积（Conditional Positional Embeddings）来保留视觉 Token 的空间位置信息。

#### 3. 实验设计
*   **数据集与场景**：涵盖了 13 个主流视觉语言基准测试，并将其分为两类：
    *   **Easy（粗粒度）**：如 ScienceQA, GQA, POPE, MMBench 等。
    *   **Hard（细粒度/高分辨率）**：如 DocVQA, InfoVQA, ChartQA, OCR-Bench 等。
*   **Benchmark 与对比方法**：
    *   **基准模型**：LLaVA-OV (0.5B, 1.5B) 和 Qwen2-VL (2B)。
    *   **对比方法**：包括训练后剪枝方法（VisionZip, VisPruner, HiRED, PyramidDrop, SparseVLM）和需要训练的压缩方法（M3）。

#### 5. 实验数量与充分性
*   **实验规模**：论文在 3 种不同规模/架构的底座模型上进行了验证，测试了超过 10 个数据集。
*   **消融实验**：非常充分。作者研究了 CA 和 SA 层数量的影响、独立训练与通用训练的对比、路由机制的有效性，以及 VISOR 与传统 Token 缩减方法结合后的增益。
*   **客观性**：通过 CKA（中心核对齐）相似度分析和层丢弃实验（Layer-drop）从理论上支撑了方法设计的合理性，实验对比均在统一的 LLaVA-OV 架构下进行，确保了公平性。

#### 6. 主要结论与发现
*   **效率提升**：VISOR 在保持甚至超过原始模型精度的前提下，实现了 **8.6倍至18倍** 的 FLOPs 节省。
*   **性能优势**：在“硬任务”（如文档理解）上，VISOR 显著优于所有 Token 缩减方法，因为它保留了完整的分辨率信息。
*   **任务依赖性**：实验证实，简单任务对视觉特征的更新需求极低，而复杂任务则需要模型在中间层不断精炼视觉表示。
*   **正交性**：VISOR 可以与 Token 缩减技术叠加使用，在极端压缩倍率（如 35倍）下仍能保持较好性能。

#### 7. 优点
*   **无损视觉信息**：避开了 Token 缩减导致的信息瓶颈，是高分辨率任务的理想方案。
*   **动态适配**：通过 Router 实现“按需分配”计算资源，比固定结构的压缩方法更智能。
*   **通用性强**：训练一次即可支持多种推理配置，且能轻易迁移到不同的 LLM 底座。
*   **理论支撑扎实**：利用 CKA 分析视觉特征演化，为稀疏交互提供了直观的解释。

#### 8. 不足与局限
*   **KV Cache 压力**：虽然减少了计算量（FLOPs），但由于保留了全部视觉 Token，推理时的显存占用（KV Cache）并未像 Token 缩减方法那样显著下降。
*   **路由策略依赖**：Router 的训练依赖于离线生成的伪标签，如果测试样本与训练分布差异极大，可能会出现配置选择失误。
*   **架构微调**：相比于纯“插件式”的训练后剪枝方法，VISOR 需要对模型进行两阶段微调，增加了一定的训练成本。
