## 摘要
近期的多模态大语言模型计算开销巨大，因为 Transformer 必须处理大量的视觉 Token。我们提出了 ReDiPrune，这是一种在视觉-语言投影器（vision-language projector）之前应用的无需训练的 Token 剪枝方法，此时视觉特征仍保持着丰富的判别性。与在压缩表示上运行的投影后剪枝方法不同，ReDiPrune 直接从视觉编码器输出中选择信息丰富的 Token，从而保留了细粒度的空间和语义线索。每个 Token 都由一个轻量级规则进行评分，该规则共同考虑了文本条件相关性和最大最小多样性，确保所选 Token 既与查询相关又非冗余。ReDiPrune 完全是即插即用的，不需要重新训练或修改架构，并且可以无缝插入到编码器和投影器之间。在四个视频和五个图像基准测试中，它一致地改善了准确率与效率之间的权衡。例如，在 EgoSchema 基准测试中使用 LLaVA-NeXT-Video-7B，仅保留 15% 的视觉 Token 即可获得 +2.0% 的绝对准确率提升，同时将 TFLOPs 计算量减少 6 倍以上。代码可在 https://github.com/UA-CVML/ReDiPrune 获取。

## Abstract
Recent multimodal large language models are computationally expensive because Transformers must process a large number of visual tokens. We present \textbf{ReDiPrune}, a training-free token pruning method applied before the vision-language projector, where visual features remain rich and discriminative. Unlike post-projection pruning methods that operate on compressed representations, ReDiPrune selects informative tokens directly from vision encoder outputs, preserving fine-grained spatial and semantic cues. Each token is scored by a lightweight rule that jointly consider text-conditioned relevance and max-min diversity, ensuring the selected tokens are both query-relevant and non-redundant. ReDiPrune is fully plug-and-play, requiring no retraining or architectural modifications, and can be seamlessly inserted between the encoder and projector. Across four video and five image benchmarks, it consistently improves the accuracy-efficiency trade-off. For example, on EgoSchema with LLaVA-NeXT-Video-7B, retaining only 15\% of visual tokens yields a +2.0\% absolute accuracy gain while reducing computation by more than $6\times$ in TFLOPs. Code is available at https://github.com/UA-CVML/ReDiPrune.

#### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLMs）在处理高分辨率图像或长视频时，视觉编码器会产生数以千计的视觉 Token。由于 Transformer 注意力机制具有二次方计算复杂度，这些冗余的视觉 Token 导致了巨大的计算开销、高内存占用和推理延迟。
*   **现有挑战**：目前的剪枝方法大多在“投影后”（Post-projection）进行，此时视觉特征已与语言空间对齐并被压缩，容易丢失细粒度的空间和语义细节；或者需要昂贵的重新训练过程。
*   **核心目标**：提出一种**无需训练、即插即用**的剪枝方案，在视觉特征进入投影器之前进行筛选，以保留最原始、最丰富的判别性信息。

#### 2. 方法论：核心思想与技术细节
ReDiPrune 的核心思想是在**投影前（Pre-projection）**阶段，基于**文本相关性**和**视觉多样性**双重准则筛选 Token。

*   **核心流程**：
    1.  **查询向量构建**：从用户 Prompt 中提取文本 Token，通过加权聚合（如指数加权）生成一个归一化的查询向量 $q$。
    2.  **相关性评分（Relevance）**：计算视觉编码器输出的每个 Token 与查询向量 $q$ 的余弦相似度 $s_i$，识别出对回答问题潜在有用的区域。
    3.  **多样性度量（Diversity）**：计算 Token 之间的余弦不相似度 $D_{ij}$，以避免选入空间上高度冗余的重复信息。
    4.  **统一目标函数**：采用贪心算法求解以下目标：$\arg \max (\min D_{ui} + \alpha s_i)$。即在保证新加入 Token 与已选集合 $S$ 具有最大差异性的同时，兼顾其与文本的相关性。
    5.  **候选预过滤**：引入阈值 $\tau$ 过滤掉低相关性 Token，进一步提升效率。
*   **即插即用**：该模块直接插入视觉编码器与投影器之间，剪枝后的少量 Token 再进入后续的 LLM 处理。

*   **实验规模**：涵盖了 9 个主流基准测试，跨越视频和图像两大领域，实验组数充足。
*   **消融实验**：针对查询向量的加权策略（均匀、自注意力、指数加权等）进行了详细对比，验证了指数加权的有效性。
*   **客观性**：通过 TFLOPs、延迟、显存占用等硬指标量化效率提升，并提供注意力图可视化（Attention Map）来解释为何剪枝能提升性能（去噪效应）。对比实验均在统一的 Token 保留率（如 10% 或 15%）下进行，确保了公平性。

#### 6. 主要结论与发现
*   **性能不降反升**：在 EgoSchema 任务中，仅保留 15% 的 Token，准确率反而提升了 2.0%。这表明剪枝起到了“去噪”作用，帮助模型聚焦关键信息。
*   **极高的压缩比**：在视频任务中，保留 10%-15% 的 Token 即可达到或超过原始模型的表现，计算量减少 6 倍以上。
*   **投影前剪枝的优越性**：相比投影后剪枝，ReDiPrune 能更好地保留细粒度语义，尤其在处理需要精确视觉定位的任务时表现更佳。

#### 7. 优点（亮点）
*   **位置前瞻**：率先提出在投影器之前进行相关性-多样性剪枝，保护了原始视觉特征。
*   **完全通用**：无需任何参数微调，适用于任何基于投影器架构的 MLLM。
*   **双重准则**：结合了文本引导（找得准）和多样性约束（看得全），有效解决了 Token 冗余问题。

#### 8. 不足与局限
*   **架构限制**：目前主要在 LLaVA 系列架构上验证，对于采用交叉注意力（Cross-attention）融合机制的模型（如 Qwen-VL）适配性有待进一步研究。
*   **时空优化**：目前的剪枝主要基于单帧内进行，尚未完全实现跨帧的全局时空最优剪枝。
*   **超参数敏感性**：平衡因子 $\alpha$ 和保留率 $r$ 仍需根据任务手动设定，缺乏自适应的 Token 预算分配机制。
